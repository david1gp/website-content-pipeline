## [0.5.0] - 2026-05-29

### 🚀 Features

- *(content)* Add `contentClean` — an explicit, idempotent frontmatter canonicalizer that rewrites source `.md` only when the normalized output differs (`ContentCleanResult` summary).
- *(sync)* `contentProcess` now auto-retries a failed bisync once with `--resync` to re-establish a missing/stale baseline, then continues. Skipped when the run already requested `--resync`.

### 🐛 Bug Fixes

- *(content)* `contentProcess` no longer rewrites source `.md` files. Frontmatter is normalized in memory for the generated `contentList.ts`; the source dir is read-only, so rclone bisync no longer safety-aborts ("all files changed on Path1") and real edits/deletes propagate again.
- *(content)* Public content copy now copies the raw source and only when missing/changed.
- *(images)* `writeImagePromptFile` only writes when the prompt content changes, stopping git churn from per-machine absolute paths.

### ⚠️ Breaking / Behavior

- `contentProcess` is now a pure reader of source content. To canonicalize frontmatter, run `contentClean` explicitly (e.g. a `content:clean` script), then `content:process`.
